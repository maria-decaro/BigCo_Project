# Homework 5: Designing and Implementing a Multi-Agent System Using Agentic AI

In this assignment, you will design, implement, and evaluate a multi-agent system (MAS) for a real-world application.  The goal is not only to build a working system, but also to understand the inner workings of complex MASs, MAS communication and coordination, and design choices for MAS as well as how to analyze and improve multi-agent workflows. A strong project for this assignment is not necessarily the biggest or most complex one. A smaller system with clear roles, thoughtful design, and clear analysis is better than a large system that is difficult to explain or evaluate. The <b>learning objectives</b> include:
* Identify a real-world problem that benefits from task decomposition
* Design a system with multiple specialized agents
* Implement an end-to-end multi-agent workflow
* Evaluate both successful and unsuccessful outputs
* Reflect on the strengths, weaknesses, and limitations of your design. 

What Counts as a Multi-Agent System? For this homework, a MAS must include at least 3 distinct agents that: 
* Have clearly different roles or responsibilities
* Exchange information through messages, shared memory, or structured outputs
* Contribute meaningfully to the final result. 

The agents may use the same underlying LLM model, as long as each agent has a clearly defined purpose. Agent roles could include: LLM models, tool-usage, rule-based, or a combination of them. A single prompt that asks one model to do everything does not count as a multi-agent system.
